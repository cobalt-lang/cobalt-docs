# Enums and Types
Enums are immutable tables of values.

```js
enum("State", {
    "Solid": 0,
    "Liquid": 1,
    "Gas": 2
})

State.Solid // 0
State.Liquid // 1
State.Gas // 2

State.Solid = 3 // Error since enums are immutable
```

they are good for representing a set of values that are known at compile time.

## Types
Types are pretty much tables. It just adds a __type file for 

```js
typedef("Person", {
    name: "string",
    age: "number"
})

newPerson = Person // You dont need to use a .new() function just reference the type
type(newPerson) // Person
```
