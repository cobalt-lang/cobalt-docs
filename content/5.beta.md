#title
New Features

#root
:ellipsis{left=0px width=40rem top=10rem blur=140px}

#default
   ::card{}
   #title
   Portability
   #description
   We are going to add more platform support, like WebAssembly, Android, Swift bridges, and more.
   ::

   ::card{}
   #title
   Standalone Executables
   #description
   cobalt24 will allow you to create and distribute standalone executables, which can be run without cobalt24 installed.
   ::

   ::card{}
   #title
   Performance
   #description
   cobalt24 will introduce many performance improvements, including a better JIT compiler, a new garbage collector, and more.
   ::

   ::card{}
   #title
   Extended Standard Library
   #description
   cobalt24 will introduce a larger standard library.
   ::

   ::card{}
   #title 
   Decorators
   #description
   Decorators will be introduced allowing you to use `@` to add metatables, 
   attributes, or bind to a signal.
   ::

   ::card{}
   #title
   Better Types
   #description
   A typechecker will be added so if the type doesnt follow the annotation rule it will not change, and `!` will make it error, and tables like 
   `[string, !number]` will be supported. `typedef` will also get a major upgrade.
   ::

   ::card{}
   #title
   Variable Declarations
   #description
   Variable declarations will be added, allowing you to define 
   variables like `var const x = 10` or `var ref x = y`, const will make the variable immutable, and ref will make it a to save memory. Custom variable declarations will also be allowd.
   ::

   ::card{}
   #title
   Better Output
   #description
   cobalt24 will have better traceback, more warnings, and a new library for handling gradients, colors, and positioning in the output. 
   ::
   
   ::card{}
   #title
   LLVM
   #description
   cobalt24 will experiment with using LLVM optionally for JIT.
   ::

   ::card{}
   #title
   Raptor
   #description
   A new version of Cobalt will be introduced, allowing live code editing and updating while your programming is running and many more development features for Interpreter and JIT.
   ::
 ::
